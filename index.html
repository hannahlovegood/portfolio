RAG Yunxi — a fully local RAG chain on the Flowise canvas

I first audited Flowise's three install paths: building from source is a detour for a demo; the officially recommended npx route is dead on Node 24 — the dependency tree pins an incompatible better-sqlite3, and npm exits silently behind a wall of warnings with the real error buried in debug logs; the official Docker image ran clean with zero compilation. Then I assembled the full RAG chain on the canvas: document chunking → local embeddings (nomic-embed-text, wired through Ollama's OpenAI-compatible endpoint so the stock node works unmodified) → a Faiss vector store → local hermes3 generation — zero API keys, zero external requests. The acceptance test is the part I care most about: I planted a fact that cannot exist in any training data (a library passphrase, "moss and letters"), so a correct answer can only mean retrieval actually worked — validating RAG with public knowledge is self-deception.

RAGFlow Local Lab — a heavyweight RAG engine running on one Mac with zero API keys

RAGFlow is a heavyweight five-container engine (Elasticsearch/MySQL/Redis/MinIO + core), and the official image is x86-only — I got it running on Apple Silicon with dependencies on native ARM and the core under Rosetta, wiring both chat and embeddings to local Ollama (hermes3 + bge-m3): zero API keys, zero external requests. Self-hosted · Bi-directional links · Whiteboard

Topic Workspace — materials, tasks and a whiteboard cross-referencing each other in self-hosted AFFiNE

First, the audit: AFFiNE is the 70k-star open-source Notion+Miro hybrid; building from source needs Node 20 + Yarn 4 + a Rust toolchain plus two NAPI native packages — that's the contributor path, while the official four-container Docker compose just works. On top of it I built a topic workspace: a materials doc, a task list and an infinite whiteboard all cross-referencing each other, with every page's Bi-Directional Links panel lighting up automatically. The real engineering was headless acceptance testing: the BlockSuite editor is full of traps for synthetic keyboards — the @-mention popover is a zero-size container that defaults to self-linking the current doc, pressing Enter in a reloaded session throws split-text index errors, and End stops at the visual line end, not the logical one. After isolating each failure I switched to a stable path: write the target doc's URL to the clipboard and paste — the editor converts it into a reference chip; verification then reads each chip's pageId instead of trusting title text — zero console errors in the final run. The full pitfall log is open-sourced.
