Self-hosted · Docker · Debug write-up

SearXNG private search — deploying is easy, the network is the real boss

A local deployment of the SearXNG metasearch engine from the official image — one query fans out to multiple engines, aggregated, untracked. The deploy itself is one config file; the real engineering is the network layer: container egress had to route back through the host proxy, and every engine treats a shared proxy exit IP differently — Google throws a JS wall, DuckDuckGo a CAPTCHA, Brave rate-limits. I benchmarked 11 candidate engines one by one, picked the 5 that actually return results, and published the whole debugging story.
